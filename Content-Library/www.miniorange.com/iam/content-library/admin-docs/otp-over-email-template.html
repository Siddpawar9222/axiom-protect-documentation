Push Gateway Configuration

Push Gateway Configuration in the context of Axiom Protect refers to the setup process that enables the system to send push notifications to users' devices. Push notifications are messages that pop up on a user's mobile device or desktop, offering a way to deliver timely and relevant information even when the user is not actively using the application.


Configure Push Gateway in your account:

  • Log in to the axiom-protect dashboard.
  • Go to Configuration tab and click on Gateway option. You will see Push Gateway Configuration


  • Click on Push Gateway Configuration


  • Description of Each Field:
    • Parameter Description
    Status Indicates whether the push gateway is active or inactive.
    Retry Count The number of times the system will retry sending a push message if the initial attempt fails.
    Retry Duration The time interval between each retry attempt.
    App ID The unique identifier for the application registered with Huawei Push Service.
    App Secret Key The secret key associated with the Huawei application, used for authenticating API requests.
    HWS Uri The endpoint URL for Huawei's push messaging service.
    API Key The key provided by the push service (e.g., Firebase Cloud Messaging) used to authenticate API requests.
    GCM Uri The endpoint URL for Google Cloud Messaging (now Firebase Cloud Messaging) service.
    Google Sender ID The unique identifier assigned to your Firebase project, used to identify the sender of the push notifications.
